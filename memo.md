Seuraavilla kerroilla, kun teet muutoksia:

git add .
git commit -m "kuvaava viesti"
git push

Siinä koko perusworkflow.

Jos haluat myöhemmin, voidaan:

tehdä README:hen pieni kuvaus pelistä

lisätä uusi versio pelistä (liian pieni / liian suuri -laajennus)

harjoitella, miten muutokset näkyvät commiteissa ja GitHubissa
